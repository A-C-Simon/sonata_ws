# Вокселизованный кэш: процедура после map_from_scans

**Быстрый запуск (план B):** из корня репозитория:
```bash
python scripts/run_voxelized_cache_semantickitti.py --data_path /path/to/SemanticKITTI/dataset
```
Если `map_world.npz` уже есть: `--skip_map`. В конце скрипт выведет команду для обучения с `--voxelized_cache_dir`.

---

## Зачем это нужно

- **map_from_scans** строит одну карту на последовательность (`ground_truth/XX/map_world.npz`) в мировой СК. Этого достаточно для обучения, но при каждом батче датасет делает тяжёлую работу: переводит всю карту в систему координат текущего кадра и вокселизует — из‑за этого CPU перегружен, GPU простаивает.
- **Вокселизованный кэш** — это шаг **после** map_from_scans: один раз по каждому кадру (pose + seq) считаем «карту в координатах этой позы» + вокселизацию и сохраняем маленький файл. При обучении только читаем эти файлы — без пересчёта карты и вокселизации. Обучение ускоряется, GPU загружен выше.

---

## Что предполагается на входе (после map_from_scans)

Структура датасета (SemanticKITTI):

- `{data_path}/sequences/XX/` — velodyne (`.bin`), labels (`.label`), `poses.txt`, `calib.txt`.
- `{data_path}/ground_truth/XX/map_world.npz` — одна карта на последовательность (результат map_from_scans). Внутри: массив точек в мировой СК (например ключ `points`).

Никаких отдельных «карт на каждый кадр» на этом шаге не требуется — только одна карта на seq.

---

## Алгоритм процедуры (пошагово)

Выполняется **один раз** после map_from_scans, по всем нужным последовательностям и кадрам.

### Входные параметры

- `data_path` — корень датасета (там же где `sequences/` и `ground_truth/`).
- `output_dir` — куда класть кэш (например `{data_path}/voxelized_cache`).
- `voxel_size` — шаг воксельной сетки (например 0.05), тот же, что при обучении.
- `max_points` — лимит точек на partial и на complete после субсэмплинга (например 20000).
- Список `sequences` — номера seq (например 00, 01, …, 10 для train).

### Для каждой последовательности XX

1. Загрузить общую карту один раз:  
   `map_world = np.load("{data_path}/ground_truth/XX/map_world.npz")["points"]` (или актуальный ключ из вашего map_from_scans).
2. Загрузить позы для seq: прочитать `sequences/XX/poses.txt` (и при необходимости скорректировать через `calib.txt`), получить массив 4×4 матриц, по одной на кадр.
3. Получить список кадров: по файлам в `sequences/XX/velodyne/*.bin` — для каждого файла есть индекс кадра и `scan_id` (имя без расширения).

### Для каждого кадра (pose) в последовательности XX

Для кадра с индексом `i` и `scan_id` (например `000123`):

1. **Поза и карта в СК кадра**
   - Взять матрицу позы для кадра `i`: `T_world_to_scan = inv(poses[i])`.
   - Перевести все точки карты из мировой СК в СК скана:  
     `points_scan = (T_world_to_scan @ [points_world | 1])[:, :3]`.  
   Это и есть «полная сцена» (GT complete) в системе координат этой позы.

2. **Partial (входной скан)**
   - Загрузить скан: `scan = load_bin(sequences/XX/velodyne/{scan_id}.bin)` → xyz (и при необходимости метки из `labels/{scan_id}.label`).
   - Центрировать: `scan_center = scan.mean(axis=0)`, `scan = scan - scan_center`.  
   То же смещение применить к `points_scan`: `complete = points_scan - scan_center`.

3. **Вокселизация**
   - **Partial:** по точкам `scan` (и меткам) построить воксельную сетку с шагом `voxel_size`: уникальные воксели → центры вокселей, метки — мажоритарный голос по вокселю. Получить `partial_coord`, `partial_labels`. В `precompute_voxelized_dataset.py` метки из `.label` приводятся к 20 классам (LEARNING_MAP как в SemanticKITTI), затем мажоритарный голос — так кэш совпадает с датасетом.
   - **Complete:** то же по точкам `complete` (метки нулевые). Получить `complete_coord`, `complete_labels`.

4. **Субсэмплинг**
   - Если точек больше `max_points`: случайно выбрать `max_points` индексов без повторений и оставить только их для partial и отдельно для complete. Так ограничиваем размер каждого сэмпла.

5. **Дополнительные поля**
   - Цвет из высоты (например z → RGB градиент): `partial_color`, `complete_color`.
   - Нормали: можно нулевые или простые, как в текущем датасете: `partial_normal`, `complete_normal`.
   - Сохранить `scan_center` (вектор из 3 float).

6. **Сохранение**
   - Один файл на кадр:  
     `{output_dir}/XX/{scan_id}.npz`  
   - Внутри (массивы numpy):  
     `partial_coord`, `partial_color`, `partial_normal`, `partial_labels`,  
     `complete_coord`, `complete_color`, `complete_normal`, `complete_labels`,  
     `scan_center`.  
   - Рекомендуется `np.savez_compressed(...)` для экономии места.

### Итоговая структура на диске

```
{output_dir}/
  00/
    000000.npz
    000001.npz
    ...
  01/
    ...
  10/
    ...
```

Один `.npz` — один кадр (одна поза, одна seq). Размер одного файла — порядка 0.5–1 MB (сжатый). На 10 train seq (~19k кадров) — примерно **10–20 GB**.

---

## Когда запускать

- **После** того как map_from_scans уже отработал и в `ground_truth/XX/` лежат `map_world.npz`.
- До обучения диффузии (или перед длительными экспериментами). Запуск один раз, можно в фоне или по ночам.

---

## Как используется при обучении

- В датасете (SemanticKITTI) при создании передаётся `voxelized_cache_dir={output_dir}`.
- В `__getitem__(idx)` сначала проверяется: есть ли файл кэша для этого `idx` (путь строится по seq и scan_id). Если есть — загружается соответствующий `.npz`, при необходимости применяется лёгкая аугментация (поворот/флип по координатам), данные конвертируются в тензоры и возвращаются. Никакой загрузки map_world и никакой вокселизации на лету.
- Если файла кэша нет — используется прежний «медленный» путь: загрузка map_world, преобразование в СК кадра, вокселизация в `__getitem__`.

Обучение запускается с аргументом, например:  
`--voxelized_cache_dir {data_path}/voxelized_cache`.

---

## Краткая схема пайплайна

1. **map_from_scans** (уже есть)  
   velodyne + poses + calib → одна карта на seq → `ground_truth/XX/map_world.npz`.

2. **Предрасчёт вокселизованного кэша** (эта процедура)  
   Для каждой (seq, кадр): map_world + поза → карта в СК кадра; скан + вокселизация partial/complete; субсэмплинг; сохранение одного маленького `.npz` на кадр.

3. **Обучение**  
   Датасет с `voxelized_cache_dir` только читает готовые `.npz` → быстрая подача батчей, выше загрузка GPU.

Логика и объём данных рассчитаны так, чтобы «тяжёлую» работу (карта в координатах каждой позы + вокселизация) делать один раз, а не на каждом батче.
